# Partner Logo Matrix Section

Create a customer logos section as one centered vertical stack inside a page-width container with comfortable side padding.

The **header block** sits at the top of the stack, centered horizontally, and contains in order from top to bottom: a primary heading on its own line; one supporting paragraph directly beneath the heading, held at a restrained max width; and below the paragraph a single horizontal row of two text links sitting side by side with a small gap between them (each link label includes a trailing arrow character at the end of the wording).

Leave substantial vertical space beneath the header block, then present a single rectangular grid of exactly twelve partner cells arranged in three rows and four columns, centered as a whole and capped at a comfortable max width that keeps each cell legible. Lay the grid out so adjacent cells share continuous horizontal and vertical dividing rules, reading as one aligned matrix rather than detached floating cards.

Inside **every cell**, stack three pieces in order from top to bottom: the partner mark centered horizontally within the cell; a single short metadata line directly under the mark providing partnership context (for example how long the relationship has run); and at the bottom of the cell a compact control for opening the partner's site whose label text is paired with a small external-link affordance icon to the right. Use the same stacking order, alignment, and internal padding in each cell so rows and columns line up cleanly across the matrix.

On narrow viewports reflow the matrix into fewer columns (for example two columns or one) while preserving reading order left-to-right then top-to-bottom. If collapsing to a single column would crowd the per-cell stack, introduce horizontal scrolling for the grid container only. Keep the header block centered above the grid and allow the link row to wrap onto a second line when space is tight.
