# Help Hub Contact Page

Create a composite contact page made of two stacked regions inside a centered page container constrained to a 1280px width with comfortable side padding.

**Upper region** (help hub) is a single centered vertical stack containing, in order: a prominent question-style heading centered on its own line; a search input field constrained to 640px width and centered horizontally beneath the heading (the input carries a placeholder inviting the user to search for a topic and an optional leading magnifying-glass affordance); then a four-column grid of categorized shortcut links on large breakpoints — each column carries a bold category label on its own line (billing and plans, support, product usage, and content or programming highlights) followed by several anchor rows stacked vertically beneath the label, each anchor on its own line.

Close the upper region with a narrow callout band that spans the full content width, framed by a thin horizontal rule on the top edge and another thin horizontal rule on the bottom edge, with 32px padding on all four sides inside the band and 32px margin above and below the band so it separates cleanly from the link grid above and the next region below. The band background is transparent. Inside the band, use a single horizontal flex row on wide breakpoints: the left side is a left-aligned vertical stack of two lines (a subheading asking if results were off-target on the first line, and a short supporting sentence pointing to the full help catalog on the second line); the right side holds a single button labeled "view all topics," end-aligned on the same row as the text block and vertically centered relative to it. On narrow viewports stack the text block above the button (left-aligned) while keeping the same top and bottom border rules and internal padding.

**Lower region** (persistent human help) carries a section heading on its own line, then splits into two columns on large 1280px width breakpoints.

The **left column** is a left-aligned vertical stack containing, in order: a headquarters-style address block (bold block title on its own line, full multi-line postal address beneath it); a sales email block (bold block title on its own line, single mailto address beneath it); a support email block (bold block title on its own line, optional short descriptive sentence, then a mailto address beneath it); a verification-of-employment email block if relevant (same shape); then a subheading for global offices on its own line followed by a short vertical list of country-labeled addresses (each entry pairing a country name and a postal address on consecutive lines).

The **right column** is a left-aligned vertical stack containing an extended form card with rounded corners. Inside the card, stack from top to bottom: a labeled email input row (the label explains that this address is used for replies; a tiny required-field marker sits to the right of the label; the input carries a standard placeholder); a labeled single-select for topic (the label sits above with a required-field marker; the select shows a default prompt); a labeled subject input row in the same shape directly below; a labeled multi-line message textarea directly below that; a single checkbox row containing a small square checkbox at the left and an inline label whose text confirms agreement to the terms of service (with the terms phrase rendered as an inline text link); one full-width primary submit button beneath the checkbox row whose label combines short send text and a trailing send-arrow glyph.

On narrow viewports, stack the four-column link grid as a two-column or single-column layout, and stack the lower region's two columns vertically with the contact-blocks column first and the form card second.
