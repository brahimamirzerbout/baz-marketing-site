# Centered Help Contact Section

Create a contact-oriented section as a single centered vertical stack constrained to a comfortable reading max width (roughly 640px) and centered horizontally with generous side padding.

At the very top, place a main heading on its own line, left-aligned. Directly below the heading, stack two full-width single-select controls (for example styled dropdowns) one above the other with a small vertical gap between them: the first control allows the user to pick a high-level category; the second control allows the user to pick a more specific sub-topic within that category.

Below the select controls, place a subsection heading on its own line (referencing popular guides and tutorials). Directly beneath the subsection heading, stack a vertical list of text links to common help articles (account access, password recovery, compromised account, activation email, notifications, login management, security options, and similar topics) — each link occupies its own line and reads as a simple anchor with no additional decoration beyond link styling.

Below the list, place a short heading on its own line that invites direct contact when nothing above matched the user's need. Under this heading, stack the inquiry form whose fields read top to bottom in this order: a labeled email input row (the label sits above the input on its own line with a tiny required-field marker; the input carries a leading envelope icon affordance flush-left inside the field and a placeholder suggesting an email address); a labeled subject input row in the same shape directly below (the label pairs with an optional small circular question-mark information affordance to the right; the input carries a placeholder inviting a short subject line); a labeled multi-line message textarea directly below that (the label pairs with an optional small circular information affordance; the textarea carries a placeholder inviting the user to describe their issue).

Under the form fields, include a paragraph-style legal note on its own line stating that submission implies agreement to linked terms and conditions and a linked privacy policy explaining collection and use of personal information (both phrases rendered as inline text links within the paragraph).

End the section with one full-width primary submit button whose label references submitting the message and includes a trailing send-arrow glyph.

On narrow viewports keep the same single-column stack; let select controls and form fields stretch to full content width with comfortable side padding.
