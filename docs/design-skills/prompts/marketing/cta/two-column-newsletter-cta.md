# Two-Column Newsletter CTA

Create a CTA section split into two columns on large 1280 px width breakpoints inside a centered page container with comfortable side padding. Align both columns to the top so the heading in the left column sits level with the top edge of the first benefit card in the right column.

The **left column** is a left-aligned vertical stack with no card chrome (no border, no background fill) containing, in order from top to bottom: a large two-line primary heading on its own line; a supporting paragraph at a comfortable reading width directly beneath the heading; generous vertical space; a small required-style field label paired with a tiny circular question-mark information affordance sitting immediately to the right of the label text on the same baseline; one horizontal row containing a wide email text input (the input carries a small leading envelope icon inside the field and placeholder copy) plus a compact solid primary button whose width hugs its single short word label, placed at the trailing end of the same row **on the same line** as the input (the email field flexes to fill all remaining horizontal space while the button never wraps below the input — treat this as the global input-plus-button row rule); a small helper line of secondary text directly beneath that row.

The **right column** stacks two equal-width benefit cards separated by a small vertical gap. Each card has a thin outer border, softly rounded corners, and generous internal padding. Inside each card, the first element is a single horizontal row placing a small circular check icon at the left and a bold single-line title to the right of the icon on the same baseline. Directly below that icon-and-title row, stack a multi-line supporting paragraph that sits flush-left under the full width of the icon-and-title row (the paragraph is not indented further under the title text alone). Keep both cards the same width and let their heights flow naturally with their content.

On narrow viewports, collapse to a single column with the newsletter block first and the two benefit cards stacked beneath it in their original top-to-bottom order.
