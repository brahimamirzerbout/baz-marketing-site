# Pricing Table with Filters

Create a pricing section as one vertically stacked block inside a max-width container. At the top, place a centered heading on its own line; directly below the heading, a centered supporting paragraph at a comfortable reading width.

Below the paragraph, center a compact control row on its own line: a short question label followed by two dropdown fields sitting side by side (for example seat or user count in the first, billing cadence in the second); each dropdown shows a small leading icon or text prefix where helpful and a chevron anchored at the trailing edge.

Below the controls, place three equal-width tier columns in one horizontal row. Inside each column, stack from top to bottom: the plan name on its own line; a large dominant price with a small "/month" suffix sitting on the same baseline to the right of the amount; one full-width call-to-action button (the highlighted middle column uses the solid primary treatment while side columns use a secondary treatment); and a text link on its own line directly below the button whose label includes a small trailing arrow glyph. The middle column carries a small "Most popular" pill badge centered on its top edge and reads as the featured tier.

Directly beneath the three tier headers — without a visual break that separates the grid — continue into one wide comparison table whose columns align vertically with the three tiers above. The middle table column shares the same highlighted band as the middle tier (a continuous outer border and subtle fill running from the tier row through every table row beneath it). The table body is grouped into category sections: each section opens with a full-width category title row spanning all columns on its own line, followed by feature rows. Each feature row contains a feature label in the first column (with an optional small hint icon beside the name and an optional inline text link in parentheses after the label); the three plan columns each hold either a small positive check icon or a small negative cross icon, vertically centered within the row.

On narrow viewports the tier row stacks vertically or becomes horizontally scrollable, the control row wraps its two dropdowns vertically, and the comparison table either gains a horizontal scroll affordance or collapses into one stacked card per plan that repeats the feature list with the same icons.
