# Selectable Plans with Detail Panel

Create a pricing section as one vertically stacked block inside a max-width container. At the top, place a small label badge (pill shape) on its own line; directly below, a large left-aligned heading on its own line; directly below the heading, a supporting paragraph at a comfortable reading width.

Below the intro block, place a horizontal two-option billing- cadence control (one segment labeled "Yearly", the other labeled "Monthly") spanning a comfortable width with one segment visually selected at a time.

Under the control, split the main area into two columns on large 1280 px width breakpoints.

**Left column** is a vertical stack of several full-width selectable plan rows that behave as a radio group (exactly one row is selected at a time). Inside each row, arrange: a circular selection indicator anchored at the left; the plan name to the right of the indicator on the same baseline, with an optional small savings or promo pill directly beneath the name; and a dominant price line with a large numeric amount and a small recurring suffix (e.g. "/month") right-aligned within the row on the same baseline as the plan name — the top tier may display a textual placeholder such as "Custom" instead of a numeric price. The selected row uses a visually distinct background and a filled indicator; unselected rows use a plain surface and an empty indicator ring.

**Right column** is a single tall panel whose height aligns with the stacked plan rows. At the top of the panel, place a short label on its own line (e.g. "Includes:"); below it, a vertical list of feature labels (each row composed of a short feature label on the left and a small check icon right-aligned within the row so the marks line up vertically). The panel's list reflects the plan that is currently selected in the left column.

Below the two-column block, place a footer row spanning the full container width: a primary call-to-action button on the left (e.g. "Choose Plan") and, on the right, a compact secondary toggle paired with a single line of explanatory copy on the same baseline (e.g. renewal or auto-billing preference).

On narrow viewports, stack in reading order: the intro block, the cadence control, the plan rail, the detail panel, and the footer row; each stretches to the full content width with comfortable side padding.
