# Four Pricing Cards with Illustrated Header

Create a pricing section as one vertically stacked block inside a max-width container. The top area is split into two columns on large 1280 px width breakpoints.

**Left header column** is a left-aligned vertical stack containing, in order: a large left-aligned heading on its own line; and a supporting paragraph directly beneath the heading at a comfortable reading width.

**Right header column** holds a decorative illustration or lightweight graphic cluster that visually balances the copy (no plan prices or feature lists appear in this column). The illustration is vertically centered relative to the left-column copy.

Below the header area, place a single row of four equal-width plan cards with consistent gutters between them. Inside each card, structure the content as three distinct horizontal bands:

**Header band** — the plan name on its own line with a single- line description directly below it.

**Body band** — a dominant price line (currency symbol, large numeric amount, and a short recurring suffix such as "/mo" on the same baseline); directly below the price, a single line stating the primary usage limit or headline cap for that tier; then a vertical list of feature rows (each row composed of a small circular check icon at the left and a short feature label to its right on the same baseline) — when the list is comparatively long, insert a horizontal dotted divider between an initial group of features and the remaining rows.

**Footer band** — one full-width call-to-action button whose label is left-aligned within the button, paired with a small arrow or external-link glyph anchored near the top-right inside the control.

Treat exactly one card as the featured tier so it reads more prominently than its neighbors (e.g. a distinct filled background or subtle accent border). From left to right, increase the headline usage limit and the number of feature rows so the row feels progressively richer.

On narrow viewports, stack the header columns in reading order (heading and paragraph first, illustration second), then stack the four plan cards vertically in their original left-to-right order; each card stretches to the full content width with comfortable side padding.
