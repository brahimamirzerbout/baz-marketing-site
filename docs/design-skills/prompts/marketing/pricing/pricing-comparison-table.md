# Pricing Comparison Table

Create a pricing section as one vertically stacked block inside a max-width container. At the top, place a centered heading on its own line; directly below the heading, a centered supporting paragraph at a comfortable reading width.

Below the heading block, place one wide comparison table spanning the full container width. The first row of the table is a sticky header row containing an empty top-left cell followed by one column header per plan; inside each header cell, stack from top to bottom: the plan name on its own line, the dominant price on a line directly below (large numeric amount with a small billing-cadence suffix on the same baseline), and one call-to-action button at the bottom of the cell whose label text is centered within the button.

The table body is grouped into category sections. Each section opens with a full-width category title row spanning all columns on its own line, followed by feature rows. Each feature row contains a feature label in the left-most column; for every plan column the cell holds either a small positive check icon, a small neutral dash icon, or a short text value — all vertically centered within the row.

On narrow viewports the table collapses into one stacked card per plan, each card repeating its own heading information and the full feature list with the same icons; cards stack vertically in the original column order.
