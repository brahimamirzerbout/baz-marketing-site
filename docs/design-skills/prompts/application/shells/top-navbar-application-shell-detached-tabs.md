# Top Navbar Application Shell With Detached Tabs

Create an application shell composed of a fixed top navbar and a horizontal tab bar separated from the navbar by a 48-pixel gap, with the remaining vertical space given entirely to the main content area. There is no sidebar in this layout.

**Top navbar** is pinned to the top of the viewport as a full-width horizontal bar. From left to right on the same baseline: a hamburger menu icon; a brand mark composed of a small logo icon paired with the brand name to its right; a search input field with a leading search icon and placeholder text; then grouped on the right end of the bar, a compact primary action button composed of a small leading plus icon and a short label, a small icon-only theme toggle button, a small circular user avatar, and a text logout link.

**Tab bar** sits below the navbar separated by a 48-pixel vertical gap so it clearly reads as outside the navbar, not part of it. The strip has no background color of its own — it inherits the content-area background. There is no full-width bottom border on the strip; only the individual tab underlines are visible. Tabs are centered horizontally within the strip as a group with consistent spacing between them. Each tab item is composed of a small leading icon on the left and a text label to its right, both aligned to the bottom of the strip. The currently active tab is distinguished by a bottom-edge underline accent in the brand color and a contrasting label color. Inactive tabs have a transparent bottom border.

**Main content area** fills the remaining viewport height below the tab bar. The 48-pixel gap, the tab strip, and the content area all share the same content-area background color so they read as one continuous surface. Content placeholder regions stack vertically as full-width blocks with comfortable side padding and consistent vertical spacing between them.

**Sizing rules.** The shell occupies the full viewport: it spans the viewport width edge to edge and stretches from the very top to the very bottom of the viewport with no surrounding page padding, no outer rounded frame, and no enclosing max-width container around the shell itself. Every dotted (dashed-outline) placeholder region inside the main content area is a short horizontal block approximately 240 pixels tall and never grows to absorb remaining vertical space; empty space below the last placeholder remains visible as the content-area background. Every visible action button inside the shell (primary, secondary outlined, dashed, full-width submit, and any inline composite action button with a label) shares one sizing rule: 12 pixels of horizontal padding on each side, 6 pixels of vertical padding on top and bottom, label text at 12 pixels in a medium font weight, and any small leading or trailing icon aligned to the label's vertical center. This button sizing rule does not apply to icon-only utility buttons, to text-only navigation links or tab triggers, or to user-identity buttons. Colors, borders, surfaces, and hover or focus states stay agnostic and follow the host design system.

On narrow viewports the navbar compresses to essential elements. The tab bar may become horizontally scrollable if tabs exceed the available width, with the active tab scrolled into view. Content regions stack vertically in reading order.
