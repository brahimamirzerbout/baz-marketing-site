# Top Navbar Sidebar Application Shell

Create an application shell composed of a fixed top navbar spanning the full viewport width and a fixed left sidebar below the navbar, with the remaining space to the right of the sidebar and below the navbar given entirely to the main content area.

**Top navbar** is pinned to the top of the viewport as a full-width horizontal bar. From left to right on the same baseline: a hamburger menu icon; a brand mark composed of a small logo icon paired with the brand name to its right; a search input field with a leading search icon and placeholder text; then grouped on the right end of the bar, a compact primary action button composed of a small leading plus icon and a short label, a small icon-only theme toggle button, a small icon-only notification bell button, a small circular user avatar, and a text logout link.

**Left sidebar** is a fixed-width vertical panel anchored to the left edge of the viewport below the navbar, stretching from the bottom of the navbar to the bottom of the viewport. Inside the sidebar, stack a vertical navigation list from top to bottom. Each primary navigation item is a horizontal row composed of a small leading icon on the left and a text label to its right on the same baseline. Some items include a trailing chevron icon indicating an expandable submenu; when expanded, indented child links appear directly below the parent item as plain text labels without icons. One section carries a highlight state (active color on the label and an expanded submenu visible). One navigation item includes a small colored numeric badge to the right of the label indicating a count. Below the main navigation group, separate a secondary group of utility links at the bottom of the sidebar (such as Docs, Components, Help), each following the same icon-plus-label row pattern.

**Main content area** fills the remaining space to the right of the sidebar and below the navbar. Content placeholder regions are arranged as a dashboard grid inside a centered container with comfortable padding: a full-width region at the top row; a three-column equal-width row of regions in the middle; a two-column unequal-width row (wider left, narrower right) below; and a full-width region at the bottom.

**Sizing rules.** The shell occupies the full viewport: it spans the viewport width edge to edge and stretches from the very top to the very bottom of the viewport with no surrounding page padding, no outer rounded frame, and no enclosing max-width container around the shell itself. Every dotted (dashed-outline) placeholder region inside the main content area is a short horizontal block approximately 240 pixels tall and never grows to absorb remaining vertical space; empty space below the last placeholder remains visible as the content-area background. Every visible action button inside the shell (primary, secondary outlined, dashed, full-width submit, and any inline composite action button with a label) shares one sizing rule: 12 pixels of horizontal padding on each side, 6 pixels of vertical padding on top and bottom, label text at 12 pixels in a medium font weight, and any small leading or trailing icon aligned to the label's vertical center. This button sizing rule does not apply to icon-only utility buttons, to text-only navigation links or tab triggers, or to user-identity buttons. Colors, borders, surfaces, and hover or focus states stay agnostic and follow the host design system.

On narrow viewports the sidebar collapses off-screen and becomes accessible via the hamburger icon in the navbar. The main content area expands to the full viewport width. Grid columns stack vertically in reading order.
