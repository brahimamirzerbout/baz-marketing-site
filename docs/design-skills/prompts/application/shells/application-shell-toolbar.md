# Application Shell With Toolbar

Create an application shell composed of two fixed horizontal bars stacked at the top of the viewport, with the remaining vertical space below given entirely to the main content area.

**Primary navbar** is pinned to the very top of the viewport as a full-width horizontal bar. From left to right on the same baseline: a hamburger menu icon; a brand mark composed of a small logo icon paired with the brand name to its right; a search input field with a leading search icon and placeholder text; then grouped on the right end of the bar, a row of small icon-only utility buttons (such as a theme toggle, a dark-mode toggle, and a grid-view toggle) sitting side by side on the same baseline, followed by a compact primary action button composed of a small leading plus icon and a short label, and finally a user identity block composed of a small circular avatar paired with the user's name and a trailing dropdown chevron to its right.

**Secondary toolbar** sits directly below the primary navbar as a full-width horizontal bar with a subtly different background or a thin top border to separate it from the navbar. On the left side of the bar, place a horizontal row of text navigation links sitting side by side on the same baseline with consistent spacing (one entry may include a trailing chevron icon indicating a dropdown submenu, and the row may end with a small overflow icon for additional links). On the right side of the same bar, place a horizontal row of filter or search widgets on the same baseline: a dropdown selector, a text input field, another dropdown selector, and a compact icon-only search button at the far right.

**Main content area** fills the remaining viewport height below the secondary toolbar inside a centered max-width container with comfortable side padding. Content placeholder regions stack vertically with consistent vertical spacing between them.

**Sizing rules.** The shell occupies the full viewport: it spans the viewport width edge to edge and stretches from the very top to the very bottom of the viewport with no surrounding page padding, no outer rounded frame, and no enclosing max-width container around the shell itself. Every dotted (dashed-outline) placeholder region inside the main content area is a short horizontal block approximately 240 pixels tall and never grows to absorb remaining vertical space; empty space below the last placeholder remains visible as the content-area background. Every visible action button inside the shell (primary, secondary outlined, dashed, full-width submit, and any inline composite action button with a label) shares one sizing rule: 12 pixels of horizontal padding on each side, 6 pixels of vertical padding on top and bottom, label text at 12 pixels in a medium font weight, and any small leading or trailing icon aligned to the label's vertical center. This button sizing rule does not apply to icon-only utility buttons, to text-only navigation links or tab triggers, or to user-identity buttons. Colors, borders, surfaces, and hover or focus states stay agnostic and follow the host design system.

On narrow viewports collapse the primary navbar into a mobile-friendly layout (hide or condense the search field; group utility icons behind a menu or shrink to essential ones; the user block may reduce to avatar-only). The secondary toolbar stacks or collapses: the navigation links may move into a hamburger or horizontal scroll strip, and the widget row may stack below the links or collapse behind a filter toggle. Preserve reading order throughout.
