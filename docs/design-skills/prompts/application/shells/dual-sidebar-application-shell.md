# Dual Sidebar Application Shell

Create an application shell composed of a fixed left sidebar spanning the full viewport height, a fixed right sidebar spanning the full viewport height, and the main content area filling the horizontal space between the two sidebars. There is no top navbar in this layout; the brand mark lives inside the left sidebar.

**Left sidebar** is a fixed-width vertical panel anchored to the left edge of the viewport, stretching the full viewport height. At the very top of the sidebar, place a brand mark composed of a small logo icon paired with the brand name to its right on the same baseline. Below the brand mark, stack a vertical navigation list. Each primary navigation item is a horizontal row composed of a small leading icon on the left and a text label to its right on the same baseline. Some items include a trailing chevron icon indicating an expandable submenu; when expanded, indented child links appear directly below the parent item. One section carries a highlight state (active color and expanded submenu). One navigation item includes a small colored numeric badge to the right of the label. Below the main navigation group, separate a secondary group of utility links at the bottom of the sidebar (such as Docs, Components, Help), each following the same icon-plus-label row pattern.

**Right sidebar** is a fixed-width vertical panel anchored to the right edge of the viewport, stretching the full viewport height. At the top of the sidebar, center a user profile block as a vertical stack: a large circular user avatar centered horizontally on its own line; the user's full name centered on its own line directly below the avatar; the user's email centered on its own line directly below the name; a full-width outlined secondary button with a leading logout icon and a short label directly below the email. Below the profile block, center a small horizontal row of icon-only utility buttons (such as a grid toggle, a settings gear, and a help circle) on the same baseline. Below the utility row, stack a vertical navigation list specific to the user's profile context. Each item is a horizontal row composed of a small leading icon on the left and a text label to its right. One section carries a highlight state and is expanded to show indented child links (such as task-status sub-items like To do, In progress, Completed).

**Main content area** fills the remaining horizontal space between the two sidebars. Content placeholder regions stack vertically as full-width blocks with consistent vertical spacing between them.

**Sizing rules.** The shell occupies the full viewport: it spans the viewport width edge to edge and stretches from the very top to the very bottom of the viewport with no surrounding page padding, no outer rounded frame, and no enclosing max-width container around the shell itself. Every dotted (dashed-outline) placeholder region inside the main content area is a short horizontal block approximately 240 pixels tall and never grows to absorb remaining vertical space; empty space below the last placeholder remains visible as the content-area background. Every visible action button inside the shell (primary, secondary outlined, dashed, full-width submit, and any inline composite action button with a label) shares one sizing rule: 12 pixels of horizontal padding on each side, 6 pixels of vertical padding on top and bottom, label text at 12 pixels in a medium font weight, and any small leading or trailing icon aligned to the label's vertical center. This button sizing rule does not apply to icon-only utility buttons, to text-only navigation links or tab triggers, or to user-identity buttons. Colors, borders, surfaces, and hover or focus states stay agnostic and follow the host design system.

On narrow viewports both sidebars collapse off-screen. The left sidebar becomes accessible via a hamburger icon. The right sidebar becomes accessible via the user avatar or a dedicated toggle. The brand mark moves into a mobile header bar. The main content area expands to the full viewport width and regions stack vertically.
