# Left Sidebar Application Shell With Toast

Create an application shell composed of a fixed left sidebar spanning the full viewport height and a top navbar that spans the remaining width to the right of the sidebar, with the main content area filling the remaining space below the navbar and to the right of the sidebar.

**Left sidebar** is a fixed-width vertical panel anchored to the left edge of the viewport, stretching the full viewport height. At the very top of the sidebar, place a brand mark composed of a small logo icon paired with the brand name to its right on the same baseline. Below the brand mark, stack a vertical navigation list. Each primary navigation item is a horizontal row composed of a small leading icon on the left and a text label to its right on the same baseline. Some items include a trailing chevron icon indicating an expandable submenu; when expanded, indented child links appear directly below the parent item. One section carries a highlight state (active color). One navigation item includes a small colored numeric badge to the right of the label. Below the main navigation group, a floating toast notification card is anchored to the lower-left area of the sidebar, containing a small leading info icon paired with a bold title on the same baseline, a close button on the right, a short multi-line description paragraph below the title, and a full-width primary action button at the bottom of the card. Beneath the toast (or beneath the main navigation if the toast is dismissed), a bottom utility group of icon-and-label links is anchored to the bottom of the sidebar (such as Settings, Help & Terms, Invite teammates).

**Top navbar** sits at the top of the viewport to the right of the sidebar as a horizontal bar. From left to right on the same baseline: a hamburger menu icon (for collapsing the sidebar); a search input field with a leading search icon and placeholder text; then grouped on the right end of the bar, a compact primary action button composed of a small leading plus icon and a short label, a small icon-only theme toggle button, a small icon-only notification bell button, a small circular user avatar, and a text logout link.

**Main content area** fills the remaining space below the navbar and to the right of the sidebar. Content placeholder regions stack vertically as full-width blocks with consistent vertical spacing between them.

**Sizing rules.** The shell occupies the full viewport: it spans the viewport width edge to edge and stretches from the very top to the very bottom of the viewport with no surrounding page padding, no outer rounded frame, and no enclosing max-width container around the shell itself. Every dotted (dashed-outline) placeholder region inside the main content area is a short horizontal block approximately 240 pixels tall and never grows to absorb remaining vertical space; empty space below the last placeholder remains visible as the content-area background. Every visible action button inside the shell (primary, secondary outlined, dashed, full-width submit, and any inline composite action button with a label) shares one sizing rule: 12 pixels of horizontal padding on each side, 6 pixels of vertical padding on top and bottom, label text at 12 pixels in a medium font weight, and any small leading or trailing icon aligned to the label's vertical center. This button sizing rule does not apply to icon-only utility buttons, to text-only navigation links or tab triggers, or to user-identity buttons. Colors, borders, surfaces, and hover or focus states stay agnostic and follow the host design system.

On narrow viewports the sidebar collapses off-screen and becomes accessible via the hamburger icon. The brand mark moves into the navbar. The toast notification repositions to the bottom of the viewport or inside a slide-out drawer. The main content area expands to the full width and regions stack vertically.
