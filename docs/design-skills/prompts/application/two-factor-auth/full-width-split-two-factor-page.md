# Full-Width Split Two-Factor Page

Create a two-factor authentication page split into two columns on large 1280px width breakpoints that spans the full available width of the viewport with no outer margin.

**Left column** uses a plain page background and contains a left-aligned vertical stack constrained to a comfortable reading max width with comfortable inset padding, vertically centered within the column height. Stack from top to bottom: a large bold heading on its own line stating the purpose (e.g. "Two-factor authentication"); a supporting paragraph of one or two sentences directly below the heading at a comfortable reading width, instructing the user to open their authenticator app and verify their identity; a labeled field group placed below the paragraph after extra vertical space, composed of a short left-aligned label on its own line (e.g. "Enter verification code") above a horizontal row of six equally sized square single-character numeric input fields on the same baseline (each field is a real input element that accepts exactly one digit, auto-advances focus to the next field on entry, supports backspace to move focus back to the previous field, with a small horizontal separator mark between the third and fourth fields to visually group the code into two halves, and the currently focused field displays a visible focus ring or cursor indicator); a short helper line directly below the input row at a subdued weight explaining where the code was sent; one primary action button placed below the helper text on its own line, composed of a small leading checkmark icon paired with a short label (e.g. "Verify account"); a recovery line placed below the button after extra vertical space, composed of a short sentence asking if the user cannot access their device, followed by an inline text link for account recovery on the same line.

**Right column** uses a distinct filled background that spans the full column area edge-to-edge, with a very subtle geometric wave or line texture layered behind the content. Inside the column, place a vertical stack of three resource cards with consistent spacing between them, vertically centered within the column height. Each card is a left-aligned vertical stack composed of a small rounded-square icon container at the top (holding a single centered glyph); a bold title on its own line directly below the icon; a short two-line description paragraph directly below the title; and a text-style link on its own line below the paragraph whose label includes a trailing arrow icon at the end of the link text.

On narrow viewports stack the two columns vertically with the left column first and the resource column directly below it; each internal stack preserves its original reading order with comfortable side padding.
