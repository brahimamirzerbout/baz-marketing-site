# Collections Application Sidebar

Create an application sidenav as a fixed-width vertical panel anchored to one edge of the viewport, stretching the full viewport height. Do not generate any main content area or placeholder blocks beside the sidenav; only output the sidenav code.

**User identity header.** At the very top of the panel, place a user identity dropdown trigger composed of a medium circular avatar on the left, a two-line text group to its right showing the currently active user (full name on the first line in a prominent weight and email on the second line in a lighter weight), and a small trailing up-down chevron icon on the far right. The entire row acts as a single clickable control with 8 pixels of internal padding so the hover state does not touch the panel edges, and opens a dropdown menu listing additional user accounts to switch between. Each dropdown item is composed of a small circular avatar paired with a two-line text group (name and email) on the same baseline as the header layout. The currently active account is visually distinguished within the menu. Selecting an item updates the active user shown in the header and closes the menu.

**Action row.** Below the user identity header, place a horizontal row containing a secondary outlined button with a small leading plus icon and a short label (such as "Add new") occupying most of the row width, and a small icon-only search button on the far right of the same baseline. The row carries a single bottom border with 16 pixels of padding below the content and 16 pixels of margin below the border, separating it from the primary navigation list. Do not add any additional top border on the navigation list — one divider line only.

**Primary navigation list.** Below the action row, stack a vertical list of navigation items with no top or bottom border of its own — only comfortable vertical spacing separating it from the action row above and the collections band below. Each primary item is a horizontal row composed of a small leading icon on the left and a text label to its right on the same baseline (such as Activity, Projects, Tasks, Contacts). One item includes a trailing chevron icon indicating a collapsible dropdown submenu.

**Collections section.** Below the primary navigation list, place the collections area inside a full-width band that is the only section in this layout carrying both a top border and a bottom border, separated from the navigation above by 16 pixels of margin on top and from content below by 16 pixels of margin on bottom, with 16 pixels of padding on top and bottom inside the bordered area. At the top of the band, place a section header row composed of a bold section title (such as "Collections") on the left and a small icon-only search button on the far right on the same baseline. Below the header, stack a vertical list of collection items. Each collection item is a horizontal row composed of a small leading icon on the left and a collection name label to its right on the same baseline (such as Sales, Design, Fundraising, Internal, Customer Success, Network, Legal). Below the collection list, place a secondary outlined button composed of a small leading plus icon and a short label (such as "Add collection") spanning the full panel width.

**Interaction behavior.** The user identity dropdown opens a menu of available accounts; selecting an account switches the active user shown in the header. The add-new button triggers the primary creation action. The search button in the action row opens a search interface. The collections search button filters the collection list. The add-collection button triggers collection creation flow.

**Sizing and layout rules.** The sidenav panel height is exactly 100 vh, always spanning the full viewport height with no gap above or below. The panel width is fixed. Content scrolls vertically if it overflows.

On narrow viewports the sidenav collapses off-screen and becomes accessible via an external toggle. Preserve reading order throughout.
