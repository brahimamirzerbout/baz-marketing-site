# Calendar Timeline Sidebar

Create an application sidenav as a fixed-width vertical panel anchored to one edge of the viewport, stretching the full viewport height. Do not generate any main content area or placeholder blocks beside the sidenav; only output the sidenav code.

**Date navigation header.** At the very top of the panel, place a date navigation row with comfortable vertical padding. Center a bold date label (such as "Tue, Sep 18") in the middle. On the left and right, place small circular icon-only previous and next arrow buttons with a light neutral background, each containing a chevron icon, all on the same baseline as the date label.

**Hourly timeline.** Below the date header, stack a vertical timeline showing consecutive hourly time slots from early morning through late evening (such as 06:00 am through 21:00 pm). Each time slot is a horizontal row composed of a short time label right-aligned in a fixed left column (such as "06:00 am") and a thin horizontal rule extending across the remaining panel width to mark the hour boundary. Empty slots show only the time label and divider with no event content.

**Event cards.** Certain time slots contain event cards placed directly below the hour divider within that slot's row area, indented from the time column. Each event card is a rounded rectangular container with a soft tinted background in a semantic accent family, a thin outer border slightly darker than the background, and a thick solid left-edge accent border (approximately 4 pixels) in the saturated accent color of the same family. Inside the card, the first line is a horizontal row composed of a small leading location-pin or clock icon paired with a time or time-range label in a medium weight on the same baseline, both rendered in the accent color (such as "08:15 am" or "10:25 am - 11:00 am"). The second line is the event title in a bold weight using a much darker shade of the same accent family for high contrast (such as "Video presentation for Flowbite AI application" or "Meeting with Bonnie Green and Jese Leos @Flowbite"). Assign each card one semantic accent such as primary, secondary, success, danger, warning, or pink to differentiate event categories — every element of the card (background tint, left border, icon, time label, and title) must stay within that single accent family. Event cards span the content width to the right of the time column with generous internal padding and consistent horizontal inset from the panel edge.

**Interaction behavior.** The previous and next arrow buttons navigate to adjacent days, updating the date label and timeline content. Event cards may be clickable to open event detail views. Empty time slots may be clickable to trigger event creation at that time.

**Sizing and layout rules.** The sidenav panel height is exactly 100 vh, always spanning the full viewport height with no gap above or below. The panel width is fixed. Time labels are right-aligned within the left time column at a consistent width. Event cards use rounded corners (approximately 8 pixels), a thick left accent border, and consistent horizontal padding from the panel edges. Populate the timeline with multiple events across the day so the list exceeds viewport height and scrolls vertically within the panel, while the date navigation header remains sticky at the top.

On narrow viewports the sidenav collapses off-screen and becomes accessible via an external toggle. Preserve reading order throughout.
