# Members and Groups Application Sidebar

Create an application sidenav as a fixed-width vertical panel anchored to one edge of the viewport, stretching the full viewport height. Do not generate any main content area or placeholder blocks beside the sidenav; only output the sidenav code.

**Members section.** At the very top of the panel, place a section header row composed of a bold title (such as "Members") on the left, a short numeric count in a lighter weight directly after the title (such as "3,456"), and a small icon-only overflow menu button (three dots) on the far right, all on the same baseline. The header carries a bottom border separating it from the list below. Below the header, stack a vertical list of member items with consistent vertical spacing. Each member item is a horizontal row composed of: a small square checkbox with rounded corners on the far left; a small circular avatar to the right of the checkbox; and a member name label to the right of the avatar in a prominent weight, all vertically centered on the same baseline. Below the member list, place a text link (such as "Show more...") in the brand accent color on its own line with comfortable padding above it.

**Groups section.** Below the members section, place a section header row composed of a bold title (such as "Groups") on the left, a short numeric count in a lighter weight directly after the title (such as "56"), and a small icon-only overflow menu button on the far right, all on the same baseline. The header carries a bottom border separating it from the list below. Below the header, stack a vertical list of group items. Each group item is a horizontal row composed of: a small square checkbox with rounded corners on the far left; a small group avatar cluster to the right of the checkbox formed by two overlapping circular thumbnails; and a group name label to the right of the avatar cluster in a prominent weight, all vertically centered on the same baseline (such as Flowbite Dev, Design Team, React Devs, Finance Team, Funny Weekend). Below the group list, place a "Show more..." text link in the brand accent color on its own line.

**Upgrade prompt.** Anchored to the very bottom of the panel, separated from the scrollable content above by a top border, place an upgrade block with comfortable padding. The block contains: a horizontal row with a plan label (such as "Free trial") on the left and a days-remaining label (such as "16 days left") on the far right in a lighter weight, on the same baseline; a horizontal progress bar below the labels with a filled segment in a success or brand accent color representing time elapsed and a neutral track for the remainder; and a full-width secondary outlined button below the progress bar with a small leading rocket or upgrade icon and a short label (such as "Upgrade to pro").

**Interaction behavior.** Checkboxes allow multi-selection of members or groups. The overflow buttons open context menus. "Show more..." links expand the lists. The upgrade button navigates to the billing or upgrade flow.

**Sizing and layout rules.** The sidenav panel height is exactly 100 vh, always spanning the full viewport height with no gap above or below. The panel width is fixed. Checkboxes, avatars, and labels are vertically centered within each row. The upgrade block is pinned to the bottom and does not scroll. Member and group lists scroll if they overflow.

On narrow viewports the sidenav collapses off-screen and becomes accessible via an external toggle. Preserve reading order throughout.
