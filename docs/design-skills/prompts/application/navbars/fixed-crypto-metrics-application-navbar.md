# Crypto Metrics Application Navbar

Create an application navbar composed of three fixed horizontal bars stacked at the top of the viewport, all spanning the full viewport width edge to edge with no surrounding page padding and no enclosing max-width container around the bars themselves.

**Top metrics bar** is pinned to the very top of the viewport as a slim full-width horizontal strip. On the left side, place a horizontal row of short key-value metric pairs sitting side by side on the same baseline with consistent spacing between them (each pair is composed of a short label followed by a prominently styled value, such as "ETH Gas: 60 Gwei", "24h Vol: $60,813,448,635", "Market Cap: $1,774,858,038,606"). On the right side of the same strip, place a locale and currency selector row on the same baseline: a small flag icon paired with a language label and a trailing dropdown chevron, followed by a currency code with a trailing dropdown chevron.

**Primary navbar** sits directly below the top metrics bar as a full-width horizontal bar. From left to right on the same baseline: a hamburger menu icon; a brand mark composed of a small logo icon paired with the brand name to its right; a search input field with a leading search icon and placeholder text, sized to occupy a moderate portion of the bar's width directly after the brand mark; then grouped on the right end of the bar, a compact primary action button composed of a small leading plus icon and a short label, a small icon-only theme toggle, a small icon-only notification bell, a small circular user avatar, and a text logout link.

**Breadcrumb and actions bar** sits directly below the primary navbar as a full-width horizontal bar with a thin top border or subtle background shift to visually separate it from the primary navbar. On the left side of the bar, place a breadcrumb row composed of icon-and-label segments separated by chevron or arrow dividers, where each segment may include a trailing dropdown chevron. On the right side of the same bar, place a row of action controls on the same baseline: a secondary outlined button composed of a small leading icon and a short label (such as "Create a team"), followed by a compact primary action button composed of a short label and a trailing plus icon (such as "New project").

**Sizing rules.** The navbar spans the full viewport width edge to edge with no surrounding page padding and no enclosing max-width container around the bars themselves. Do not generate any main content area or placeholder blocks below the navbar; only output the navbar code. Every visible action button inside the navbar (primary, secondary outlined, dashed, and any inline composite action button with a label) shares one sizing rule: 12 pixels of horizontal padding on each side, 6 pixels of vertical padding on top and bottom, label text at 12 pixels in a medium font weight, and any small leading or trailing icon aligned to the label's vertical center. This button sizing rule does not apply to icon-only utility buttons, to text-only navigation links, or to user-identity buttons. Colors, borders, surfaces, and hover or focus states stay agnostic and follow the host design system.

On narrow viewports the top metrics bar collapses into a horizontally scrollable strip or hides behind a toggle; the primary navbar compresses to essential elements (search collapses behind a search icon, the primary action becomes icon-only, user block reduces to avatar-only); the breadcrumb truncates to the last segment with a back arrow and the action controls stack vertically or collapse into an overflow menu. Preserve reading order throughout.
