# Dual-Bar Application Navbar

Create an application navbar composed of two fixed horizontal bars stacked at the top of the viewport, both spanning the full viewport width edge to edge with no surrounding page padding and no enclosing max-width container around the bars themselves.

**Primary navbar** is pinned to the very top of the viewport as a full-width horizontal bar. From left to right on the same baseline: a hamburger menu icon; a brand mark composed of a small logo icon paired with the brand name to its right on the same baseline; then grouped on the right end of the bar, a small icon-only theme toggle, a small icon-only search trigger, a small icon-only notification bell, a small circular user avatar, and a text logout link.

**Secondary navigation bar** sits directly below the primary navbar as a full-width horizontal bar with a thin top border or a subtly different background to separate it from the primary navbar. On the left side of the bar, place a horizontal row of text navigation links sitting side by side on the same baseline with consistent spacing between them. Some entries include a trailing chevron icon indicating a dropdown submenu, and the row ends with a small overflow icon for additional links.

**Sizing rules.** The navbar spans the full viewport width edge to edge with no surrounding page padding and no enclosing max-width container around the bars themselves. Do not generate any main content area or placeholder blocks below the navbar; only output the navbar code. Every visible action button inside the navbar (primary, secondary outlined, dashed, and any inline composite action button with a label) shares one sizing rule: 12 pixels of horizontal padding on each side, 6 pixels of vertical padding on top and bottom, label text at 12 pixels in a medium font weight, and any small leading or trailing icon aligned to the label's vertical center. This button sizing rule does not apply to icon-only utility buttons, to text-only navigation links, or to user-identity buttons. Colors, borders, surfaces, and hover or focus states stay agnostic and follow the host design system.

On narrow viewports the secondary navigation bar collapses into the hamburger menu or becomes a horizontally scrollable strip with the active link scrolled into view; the primary navbar compresses to essential elements (hamburger, brand, avatar, overflow). Preserve reading order throughout.
