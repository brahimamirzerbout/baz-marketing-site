# Application Navbar With Filter Toolbar

Create an application navbar composed of two fixed horizontal bars stacked at the top of the viewport, both spanning the full viewport width edge to edge with no surrounding page padding and no enclosing max-width container around the bars themselves.

**Primary navbar** is pinned to the very top of the viewport as a full-width horizontal bar. From left to right on the same baseline: a hamburger menu icon; a brand mark composed of a small logo icon paired with the brand name to its right; a search input field with a leading search icon and placeholder text, sized to occupy a moderate portion of the bar's width directly after the brand mark; then grouped on the right end of the bar, a row of small icon-only utility buttons sitting side by side on the same baseline (such as a theme toggle, a settings gear, and a grid-view toggle), followed by a compact primary action button composed of a small leading plus icon and a short label, and finally a user identity block composed of a small circular avatar paired with the user's name and a trailing dropdown chevron.

**Secondary toolbar** sits directly below the primary navbar as a full-width horizontal bar with a thin top border or a subtly different background to separate it from the primary navbar. On the left side of the bar, place a horizontal row of text navigation links sitting side by side on the same baseline with consistent spacing between them. Some entries include a trailing chevron icon indicating a dropdown submenu, and the row ends with a small overflow icon for additional links. On the right side of the same bar, place a horizontal row of filter widgets on the same baseline: a dropdown selector with a trailing chevron, a text input field with placeholder text, another dropdown selector with a trailing chevron, and a compact icon-only search button at the far right.

**Sizing rules.** The navbar spans the full viewport width edge to edge with no surrounding page padding and no enclosing max-width container around the bars themselves. Do not generate any main content area or placeholder blocks below the navbar; only output the navbar code. Every visible action button inside the navbar (primary, secondary outlined, dashed, and any inline composite action button with a label) shares one sizing rule: 12 pixels of horizontal padding on each side, 6 pixels of vertical padding on top and bottom, label text at 12 pixels in a medium font weight, and any small leading or trailing icon aligned to the label's vertical center. This button sizing rule does not apply to icon-only utility buttons, to text-only navigation links, or to user-identity buttons. Colors, borders, surfaces, and hover or focus states stay agnostic and follow the host design system.

On narrow viewports the primary navbar compresses to essential elements (search collapses behind a search icon, utility icons group behind an overflow menu, primary action becomes icon-only, user block reduces to avatar-only); the secondary toolbar stacks or collapses (navigation links move into the hamburger or a horizontally scrollable strip, and the filter widget row stacks below the links or collapses behind a filter toggle). Preserve reading order throughout.
