# Application Top Navbar

Create an application navbar as a single fixed horizontal bar pinned to the top of the viewport, spanning the full viewport width edge to edge with no surrounding page padding and no enclosing max-width container. Do not generate any main content area or placeholder blocks below the navbar; only output the navbar code.

**Primary navbar** is the only bar in this layout and contains, from left to right on the same baseline: a hamburger menu icon; a brand mark composed of a small logo icon paired with the brand name to its right on the same baseline; a search input field with a leading search icon and placeholder text, sized to occupy a moderate portion of the bar's width directly after the brand mark; then grouped on the right end of the bar, a compact primary action button composed of a small leading plus icon and a short label, followed by a row of small icon-only utility buttons (such as a theme toggle and a notification bell) sitting side by side on the same baseline, a small circular user avatar, and a text logout link at the far right.

**Sizing rules.** The navbar spans the full viewport width edge to edge with no surrounding page padding and no enclosing max-width container around the bar itself. Every visible action button inside the navbar (primary, secondary outlined, dashed, and any inline composite action button with a label) shares one sizing rule: 12 pixels of horizontal padding on each side, 6 pixels of vertical padding on top and bottom, label text at 12 pixels in a medium font weight, and any small leading or trailing icon aligned to the label's vertical center. This button sizing rule does not apply to icon-only utility buttons, to text-only navigation links, or to user-identity buttons. Colors, borders, surfaces, and hover or focus states stay agnostic and follow the host design system.

On narrow viewports collapse the navbar into a mobile-friendly layout: hide or condense the search field behind a search icon toggle; group utility icons behind a menu or shrink to essential ones; the user block may reduce to avatar-only; and the primary action button may collapse into an icon-only button. Preserve reading order throughout.
