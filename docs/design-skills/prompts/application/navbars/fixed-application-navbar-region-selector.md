# Application Navbar With Region Selector

Create an application navbar as a single fixed horizontal bar pinned to the top of the viewport, spanning the full viewport width edge to edge with no surrounding page padding and no enclosing max-width container. Do not generate any main content area or placeholder blocks below the navbar; only output the navbar code.

**Primary navbar** is the only bar in this layout and contains, from left to right on the same baseline: a hamburger menu icon on the left; a brand mark composed of a small logo icon paired with the brand name to its right on the same baseline directly after the hamburger; then grouped on the right end of the bar, a small icon-only grid-view toggle, a small icon-only notification bell, a compact search trigger composed of a small leading search icon paired with a short "Search" label on the same baseline, a region selector composed of a small flag or location glyph paired with a location label and a trailing dropdown chevron, and finally a user identity block composed of a small circular avatar paired with the user's name and a small role badge in parentheses on the same baseline. A thin vertical divider may sit between adjacent grouped elements on the right end to visually separate them.

**Sizing rules.** The navbar spans the full viewport width edge to edge with no surrounding page padding and no enclosing max-width container around the bar itself. Every visible action button inside the navbar (primary, secondary outlined, dashed, and any inline composite action button with a label) shares one sizing rule: 12 pixels of horizontal padding on each side, 6 pixels of vertical padding on top and bottom, label text at 12 pixels in a medium font weight, and any small leading or trailing icon aligned to the label's vertical center. This button sizing rule does not apply to icon-only utility buttons, to text-only navigation links, or to user-identity buttons. Colors, borders, surfaces, and hover or focus states stay agnostic and follow the host design system.

On narrow viewports the search trigger, region selector, and role badge may collapse off-screen or behind a menu toggle; the user identity block may reduce to avatar-only; utility icons may group behind an overflow menu. Preserve reading order throughout.
