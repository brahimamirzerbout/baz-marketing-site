# Split Marketing Register Page

Create a register page split into two columns on large 1280px width breakpoints inside a centered page container with comfortable side padding. The left column is a marketing column carrying copy and a short feature list; the right column is the form column. The two columns share the same vertical center line.

The **left column** is a left-aligned vertical stack containing, in order: a large multi-line heading on its own line; a supporting paragraph below the heading at a comfortable reading width; a vertical list of about three feature rows each composed of a small leading icon at the left, a bold one-line title to the right of the icon on the same baseline, and a short supporting line directly beneath the title that sits flush-left under the icon-and-title row. Allow a very subtle decorative line or wireframe pattern to sit behind the left column so it reads as a textured marketing surface without competing with the copy.

The **right column** is a left-aligned vertical stack containing, in order: a heading on its own line; a horizontal row of two equal third-party provider buttons sitting side by side (each button carries a small leading provider glyph and a short provider label); a horizontal "or" separator made of a thin rule, a centered short word, and another thin rule; a two-column field grid pairing a short text input on the left with a short text input on the right (first name / last name); a second two-column field grid pairing a short text input on the left with a single dropdown select on the right (email / country); a third two-column field grid pairing a password input on the left with a repeat-password input on the right (every label sits above its input with a tiny required-field marker to the right of the label text where applicable); one checkbox row containing a small square checkbox and an inline multi-line label whose copy may contain inline text links to legal documents; one full-width primary submit button beneath the checkbox row; finally a small helper line containing a short prompt sentence followed by an inline text link to the sign-in flow.

On narrow viewports, stack with the form column first and the marketing column second so the user reaches the form quickly; keep all internal stacks in their original reading order.
